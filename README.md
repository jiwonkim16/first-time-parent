노마드코더 *AI Agent Master Class* — LangGraph 파트 챌린지 과제로 만든 커스텀 에이전트입니다.

## 이 에이전트가 하는 일 / 하지 않는 일

| ❌ 하지 않음 | ✅ 함 |
|---|---|
| 아기 증상을 **진단** | 어떻게 대처할지 **교육** |
| 병원 갈지 대신 **결정** | 병원 상담이 필요한 **신호를 알려줌** |
| 육아 **정답** 제시 | **공식 근거**로 선택지 정리 |

의료 진단·처방을 하지 않습니다. 모든 답변 끝에 **실제 상담 연락처**(1339·1577-0756)를 함께 안내하며,
최종 판단은 항상 의료진에게 연결합니다.

## 핵심 기능

- **세션 프로필** — 아이 이름·월령을 세션에만 보관(탭 닫으면 삭제). 미성년자 민감정보 비영속 원칙.
- **위험도 판단 (보수적)** — LLM 판단과 rule guardrail을 **병렬 실행**하고 `max()`로 합쳐 안전측 채택.
- **3단계 분기** — 높음(즉시 병원)/주의(오늘 진료+교육)/낮음(7단 교육).
- **되묻기** — 위험도를 가늠할 핵심 정보가 없으면 먼저 다정하게 되묻습니다.
- **공식 근거 검색** — 월령·주제로 아이사랑·CDC·질병관리청 등 공식 자료를 조회해 인용.
- **근거 부재 정책** — 자료를 못 찾아도 침묵하지 않고 위험도별로 고지 후 안내.
- **역할 범위 밖 처리** — 육아 무관 질문은 정중히 거절.
- **세션 다중턴 맥락** — 이전 대화를 세션 메모리로 기억(월령·앞선 질문 반영). 체크포인터는 쓰지 않습니다.

## ⚠️ 이 도구의 한계

**이 앱은 자유 대화 챗봇이 아니라 '일문일답' 교육 도구입니다.**
설계상 낮음·주의 위험도 질문에는 **항상 7단 구조의 완결된 교육 답변**을 제공합니다(기획 §10).
따라서 다음과 같은 대화형 상호작용에는 최적화되어 있지 않습니다.

- "늦었다/안 늦었다로만 답해줘" 같은 **답변 형식 변경 요청**
- 앞선 답변에 대한 **반박·되묻기**에 짧게 이어 말하기
- 같은 주제를 여러 번 물으면 매번 7단 교육을 다시 제공(내용이 반복될 수 있음)

이는 초보 부모가 매 질문마다 **빠짐없는 교육**을 받도록 한 의도된 설계입니다.
자연스러운 후속 대화가 필요하다면 별도 확장(후속 응답 노드)이 필요합니다.

## 아키텍처 (LangGraph 워크플로우)

```
START → analyze ──(정보부족?)──→ clarify → END        # 되묻기
          │      ──(육아무관?)──→ out_of_scope → END
          └─ [llm_risk ‖ rule_risk] → aggregate       # 병렬 + max()
                                       │
                    high → decide                       # 즉시 병원 안내
                    warn/low → search → coach → decide  # 근거 조회 → 7단 교육
                                                → END
```

적용한 LangGraph 개념: State·Node·조건부 Edge, 병렬화+voting+aggregator,
Tool + `bind_tools` + `ToolNode` ReAct 루프(근거 검색), `with_structured_output`.

- `agent.py` — 그래프 정의 (State·노드·라우팅·근거 레지스트리)
- `app.py` — Streamlit 채팅 UI
- `test_agent.py` — 결정론적 안전 로직 테스트 (pytest)

## 로컬 실행

```bash
# 1. 의존성 설치 (uv 사용)
uv sync

# 2. OpenAI API 키 설정 — .env 파일
echo "OPENAI_API_KEY=sk-..." > .env

# 3. 앱 실행
uv run streamlit run app.py

# (선택) 로직 테스트
uv run pytest test_agent.py -vv
# (선택) 그래프 end-to-end 자가검증
uv run python agent.py
```

## Streamlit Cloud 배포

1. 이 저장소를 GitHub에 푸시합니다. (`requirements.txt` 포함)
2. [share.streamlit.io](https://share.streamlit.io) 접속 → **GitHub 연결**.
3. 저장소·브랜치 선택, Main file path에 `app.py` 지정.
4. **Advanced settings → Secrets**에 API 키를 넣습니다:
   ```toml
   OPENAI_API_KEY = "sk-..."
   ```
   > 앱은 `st.secrets` → 환경변수 순으로 키를 읽으므로 이 설정만으로 동작합니다.
5. **Deploy!**
